3.3.3.2.1. Gerenciamento

3.3.3.2.1.1. O equipamento deve possuir interface de rede dedicada para gerenciamento;

3.3.3.2.1.2. Suportar as interfaces e protocolos de gerenciamento WS- MAN, CIM, IPMI e SNMP;

3.3.3.2.1.3. Possuir software console do mesmo fabricante do servidor, com capacidade de gerenciamento remoto de um único equipamento (1:1) e vários equipamentos (1:N);

3.3.3.2.1.4. Suportar os protocolos de criptografia SSL para acesso Web e SSH para acesso CLI;

3.3.3.2.1.5. O software console deve realizar descoberta e inventário remoto dos servidores e seus componentes;

3.3.3.2.1.6. Permitir o monitoramento remoto, através do software console das condições de funcionamento dos equipamentos e seus componentes, tais como: processadores, memória RAM, controladora RAID, discos, fontes de alimentação, NICs e ventiladores;

3.3.3.2.1.7. Emitir alertas de anormalidade de hardware através do software console e suportar o encaminhamento via e-mail e trap SNMP;

3.3.3.2.1.8. Permitir o monitoramento remoto 1:1 e 1:N do consumo de energia em tempo real através do software console com exibição gráfica, permitindo gerenciar o consumo de energia elétrica dos equipamentos;

3.3.3.2.1.9. Permitir a configuração remota de parâmetros da BIOS e RAID através de console remota;

3.3.3.2.1.10. Permitir ligar, desligar e reiniciar os servidores remotamente e independente de sistema operacional;

3.3.3.2.1.11. Permitir o controle remoto do tipo virtual KVM out-of-band, ou seja, independente de sistema operacional ou software agente;

3.3.3.2.1.12. Permitir a emulação de mídias virtuais de inicialização (boot) através de CD/DVD remoto, compartilhamentos de rede NFS/CIFS e dispositivos de armazenamento USB remotos;

3.3.3.2.1.13. Suportar autenticação local e através de integração com MS Active Directory/LDAP;

3.3.3.2.1.14. Permitir a captura de vídeo ou tela de situações de falhas críticas de sistemas operacionais e inicialização do sistema (boot), possibilitando uma depuração mais aprimorada;

3.3.3.2.1.15. As funcionalidades de gerenciamento e monitoramento de hardware devem ser providas por recursos do próprio equipamento e independente de agentes ou sistema operacional;

3.3.3.2.1.16. Permitir a instalação, update e configuração remota de sistemas operacionais, drivers e firmwares através de solução de deployment compatível com a solução ofertada;

3.3.3.2.1.17. A solução deve possuir recurso que possibilite a reposição de componentes sem necessidade de reconfigurações, através da restauração da configuração de firmwares anteriores;

3.3.3.2.1.18. A solução de gerenciamento deve estar devidamente licenciada conforme as condições de garantia e suporte do equipamento;
3.3.3.2.1. Compatibilidades

3.3.3.2.1.1. Canonical Ubuntu Server LTS;

3.3.3.2.1.2. Microsoft Windows Server with Hyper-V;

3.3.3.2.1.3. Red Hat Enterprise Linux;

3.3.3.2.1.4. Suse Linux Enterprise Server;

3.3.3.2.1.5. VMware ESXi.
